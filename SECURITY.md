# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**在公开 Issue 中报告。

请发送邮件至 [security@example.com]，包含：

1. 漏洞描述
2. 复现步骤
3. 影响范围
4. 建议的修复方案（如有）

我们会在 48 小时内回复，并在修复后发布安全更新。

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 支持 |
| < 1.0  | ❌ 不支持 |

## 安全最佳实践

- 不要在代码中硬编码 API Key
- 使用环境变量管理敏感配置
- 定期更新依赖以获取安全补丁
- 在生产环境中使用 HTTPS
